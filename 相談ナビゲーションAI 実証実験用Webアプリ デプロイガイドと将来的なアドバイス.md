# 相談ナビゲーションAI 実証実験用Webアプリ デプロイガイドと将来的なアドバイス

このドキュメントでは、開発した「相談ナビゲーションAI」WebアプリをStreamlit Community Cloudにデプロイし、Web上で共有するための具体的な手順と、将来的なLGWAN環境での活用に向けたアドバイスを提供します。

## 1. Streamlit Community Cloudへのデプロイ手順

Streamlit Community Cloudは、Streamlitアプリを無料でデプロイし、Web上で公開できるサービスです。以下の手順でデプロイを進めます。

### ステップ1: GitHubリポジトリの作成

1.  **GitHubアカウントの作成**: まだお持ちでない場合は、[GitHub](https://github.com/)でアカウントを作成します。
2.  **新しいリポジトリの作成**: GitHubにログイン後、右上の「+」アイコンから「New repository」を選択します。
    -   **Repository name**: `consultation-navigation-ai` など、任意の分かりやすい名前を付けます。
    -   **Description**: 「自治体向け相談ナビゲーションAIの実証実験アプリ」など、説明を記入します。
    -   **Public/Private**: 「Public」を選択します。Streamlit Community CloudはPublicリポジトリからデプロイするのが最も簡単です。
    -   「Add a README file」にチェックを入れ、「Create repository」をクリックします。
3.  **ファイルのアップロード**: 作成したリポジトリに、以下の3つのファイルをアップロードします。
    -   `app.py` (Streamlitアプリケーションのメインコード)
    -   `requirements.txt` (必要なPythonライブラリのリスト)
    -   `consultation_data.csv` (ダミーデータ)
    
    ファイルをアップロードするには、リポジトリページで「Add file」→「Upload files」を選択し、ファイルをドラッグ＆ドロップするか、ファイル選択ダイアログから選択します。コミットメッセージを入力して「Commit changes」をクリックします。

### ステップ2: Streamlit Community Cloudでのデプロイ

1.  **Streamlit Cloudにサインアップ**: [Streamlit Community Cloud](https://streamlit.io/cloud)にアクセスし、GitHubアカウントでサインアップまたはログインします。
2.  **新しいアプリのデプロイ**: ログイン後、ダッシュボードの右上にある「New app」ボタンをクリックします。
3.  **リポジトリ情報の入力**: 以下の情報を入力します。
    -   **Repository**: 作成したGitHubリポジトリを選択します（例: `your-github-username/consultation-navigation-ai`）。
    -   **Branch**: `main` または `master` を選択します。
    -   **Main file path**: `app.py` と入力します。
    -   **App URL**: 任意のURLを設定できますが、通常は自動生成されたもので問題ありません。
4.  **デプロイ**: 「Deploy!」ボタンをクリックします。Streamlit CloudがGitHubリポジトリからコードをクローンし、`requirements.txt`に基づいて必要なライブラリをインストールし、アプリを起動します。このプロセスには数分かかる場合があります。
5.  **アプリの確認**: デプロイが完了すると、アプリのURLが表示され、Webブラウザでアクセスできるようになります。

## 2. 将来的なLGWAN環境での活用に向けたアドバイス

この実証実験用Webアプリは、インターネットに接続された環境での動作を前提としていますが、自治体の基幹業務システムが稼働するLGWAN（総合行政ネットワーク）環境下での利用には、いくつかの考慮事項と対応が必要です。

### 1. 三層分離環境への対応

多くの自治体では、セキュリティ確保のため「三層分離」と呼ばれるネットワーク構成を採用しています。これは、インターネット接続系、LGWAN接続系、個人番号利用事務系の3つのネットワークを物理的または論理的に分離するものです。

-   **インターネット接続系**: 本アプリのようにインターネット経由でアクセスするWebサービスは、このネットワークに配置されます。職員がインターネット接続系の端末から本アプリを利用することは可能です。
-   **LGWAN接続系**: 住民情報や税情報など、機密性の高い情報を取り扱う業務システムが稼働するネットワークです。原則としてインターネットへの直接接続はできません。

**アドバイス**: 

LGWAN端末から本アプリの機能を利用するためには、直接的なアクセスは困難です。以下のいずれかのアプローチが考えられます。

-   **データ連携**: LGWAN環境内の業務システム（例: Kintoneなどの業務プラットフォーム）と、インターネット接続系の本アプリとの間で、**セキュアなデータ連携基盤**を構築する必要があります。例えば、LGWAN環境から匿名化・非個人情報化された相談内容を抽出し、インターネット接続系の連携サーバー経由で本アプリに渡し、結果をLGWAN環境に戻すといった仕組みです。この際、データの暗号化、アクセス制御、監査ログの取得などが必須となります。
-   **ハイブリッドクラウド/ガバメントクラウドの活用**: 将来的には、ガバメントクラウドの利用が進むことで、LGWAN環境とインターネット接続系のサービス間の連携がよりセキュアかつ容易になる可能性があります。ガバメントクラウド上で本アプリを稼働させ、LGWAN環境からのセキュアなアクセス経路を確保することが検討されます。

### 2. LLMの利用とデータガバナンス

本アプリはLLM（大規模言語モデル）を利用するモードを搭載していますが、自治体業務でLLMを利用する際には、以下の点に特に注意が必要です。

-   **個人情報・機密情報の取り扱い**: LLMに個人情報や機密情報を入力することは、情報漏洩のリスクを伴います。LLMモードを利用する際は、**入力する相談内容から個人を特定できる情報を完全に排除**する、あるいは**機密情報をマスキング**するなどの運用ルールを徹底する必要があります。
-   **プロンプトインジェクション対策**: 悪意のあるプロンプトによってLLMが意図しない動作をしたり、機密情報を引き出されたりするリスクがあります。プロンプトの検証とフィルタリング、出力内容の監視が必要です。
-   **利用するLLMの選定**: 利用するLLMが、入力されたデータを学習に利用しないこと、データが適切に保護されることを明示しているサービスを選定することが重要です。自治体向けに特化したセキュアなLLMサービスや、オンプレミスで動作するLLMの導入も検討の余地があります。

### 3. 継続的な改善と運用

実証実験を通じて得られた知見を基に、以下の改善を継続的に行うことが重要です。

-   **データソースの拡充**: ダミーデータだけでなく、実際の業務マニュアル、FAQ、過去の相談事例などをデータソースとして取り込むことで、AIの回答精度を向上させます。
-   **検索精度の向上**: TF-IDFやコサイン類似度だけでなく、より高度な埋め込みモデル（Embedding Model）やベクトルデータベース（Vector Database）を導入することで、RAGの検索精度を大幅に向上させることができます。
-   **UI/UXの改善**: 実際に利用する職員からのフィードバックを収集し、より使いやすく、業務に即したインターフェースに改善します。
-   **セキュリティ対策**: 定期的な脆弱性診断、アクセスログの監視、最新のセキュリティパッチ適用など、継続的なセキュリティ対策が不可欠です。

この実証実験が、自治体のDX推進の一助となることを期待しています。
